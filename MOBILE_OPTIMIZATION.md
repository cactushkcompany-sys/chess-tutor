# 移动端优化说明

## 已完成的优化

### 1. 触摸体验优化
- ✅ 禁用双击缩放，防止误操作
- ✅ 移除文本选择，避免拖动棋子时选中文字
- ✅ 优化触摸高亮效果
- ✅ 所有按钮添加 `touch-manipulation` 类，提升响应速度

### 2. 响应式布局
- ✅ 棋盘自动适配屏幕宽度（最大 600px）
- ✅ 监听屏幕旋转事件，横竖屏自动调整
- ✅ 使用 Flexbox 布局，确保内容不溢出
- ✅ 文字大小响应式调整（sm: 断点）

### 3. 安全区域适配
- ✅ 支持 iPhone 刘海屏和底部手势条
- ✅ 使用 `safe-area-inset-*` 环境变量
- ✅ 添加 `viewport-fit=cover` 支持全屏显示

### 4. PWA 支持
- ✅ 添加 manifest.json
- ✅ 支持"添加到主屏幕"
- ✅ 独立窗口模式（standalone）
- ✅ 自定义主题色和图标

### 5. 性能优化
- ✅ 减少烟花粒子数量（移动端 60-150 个）
- ✅ 优化动画性能，减少重绘
- ✅ 特效文字大小响应式调整
- ✅ 使用 `will-change` 提示浏览器优化

### 6. 用户体验
- ✅ 顶部导航栏固定，防止滚动时消失
- ✅ 游戏区域可滚动，适配小屏幕
- ✅ 按钮大小适中，易于点击（最小 44x44px）
- ✅ 错误提示清晰，字体大小合适

## 已知问题

1. **iOS Safari 音效延迟**
   - 原因：iOS 限制自动播放音频
   - 解决：需要用户首次交互后才能播放

2. **部分 Android 设备烟花卡顿**
   - 原因：低端设备性能不足
   - 解决：已减少粒子数量，可进一步优化

## 开发建议

### 测试移动端
```bash
# 启动开发服务器，允许局域网访问
npm run dev -- -H 0.0.0.0

# 在手机浏览器访问
http://你的电脑IP:3000
```

### 调试工具
- **iOS**: Safari → 开发 → 你的设备
- **Android**: Chrome → chrome://inspect
- **模拟器**: Chrome DevTools → Toggle device toolbar (Cmd+Shift+M)

### 性能分析
```javascript
// 在浏览器控制台运行
performance.mark('start');
// ... 执行操作
performance.mark('end');
performance.measure('operation', 'start', 'end');
console.table(performance.getEntriesByType('measure'));
```
